# jQuery

## 1.邂逅jQuery

- 什么是jQuery,以及优点
  - 跨浏览器支持(出现的主因),以前不同浏览器要写很多兼容代码
  - 是一个javascript库
  - 易于学习,支持JavaScript编码风格
  - 少写多做(即原生JavaScript代码用jQuery几句就行了)
    - 如其提供丰富的功能(DOM操作,过滤器...)



- jQuery的缺点
  - jQuery代码库占用内存不断增大
  - 不支持组件化开发
  - 适合DOM操作,稍微复杂项目,其能力有限,这时要用到前端三大框架(vue,Angular,React)
    - 但这些框架很多都借用了jQuery的思想
    - 同时要是公司旧代码用的就是jQuery,这就是晋升机会



- jQuery的历史
  - 为了跨浏览器不写那么多兼容代码



## 2. 库和框架的区别

- 框架里面不止有库还有其他如插件



- 库是代码片段集合,如函数,变量,常见库如jQuery,Day.js

- 框架是个完整的工具集,如vue,你代码要按他给的格式写

## 4. 什么是CDN

- Content Delivery Network(内容分发网络)
- 是一组分布在不同地理位置的服务器相互连接形成的网络系统
- 通过这个系统,用户可以通过离其最近的服务器拿到事先缓存到这个服务器的web内容(如视频,图片)



- 作用
  - 不仅提高资源访问速度
    - ![]()
  - 同时分担源站压力
    - ![]()



- 购买
  - 自己购买CDN服务(如阿里,腾讯)
  - 开源的CDN服务

## 7. jQuery变量名的冲突

- jQuery.noConflict()

  - ```
     <!-- 当其他库里面的$符号冲突时，可以使用jQuery.noConflict()来解决 -->
      
      <script>  
        let $ = '我是其他库里面的$符号'
        let jQuery = '我是其他库里面的jQuery符号'
        console.log('%o', $)//我是其他库里面的$符号
        console.log(jQuery)//我是其他库里面的jQuery符号
        
        // 解决冲突
       const $new$ = window.$.noConflict(true)
        // const $newJ = window.jQuery.noConflict(true)//和上面不能同时用,会报错
        console.log($new$)//JQuery对象
        console.log(window.jQuery)//undefined
    ```
